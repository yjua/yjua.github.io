原型链的一些理解

JavaScript

原型链是JavaScript中特别重要的一个东西。

我们要知道：

  • javascript中一切皆对象（Object），对象是对象，数组、函数这些也是对象。
  • 在面向对象语言中都会通过类来实现继承，而javascript中则通过原型链来实现继承。

prototype

JavaScript中的原型链是靠原型（prototype）串联起来实现的。但不是所有的对象都拥有prototype这一属性。

prototype是每个函数才有的属性，通过该属性访问到的对象就是该函数的原型对象。原型对象是一个特殊的Object对象，它有个constructor属性，该属性又指向原型对象的构造函数。

如果函数和new这个关键词一起使用的时候，我们认为这个函数是构造函数。通过new出来的对象，我们称之为构造函数的实例化。

__proto__

在JavaScript中每个对象（undefined,null特殊对象除外）都有一个内置的[[Prototype]]属性，这是一个内部指向链接，外部无法访问，它指向其‘父类’的prototype。在chrome等现代浏览器中，提供了一个__proto__属性来指代[[Prototype]]属性。

也就是说每个对象都指向了其构造函数的原型对象（prototype），而原型对象也是一个对象，所以它也会指向它自己的构造函数的的原型对象，以此类推，最终会指向一个null对象，而null是这个链条的终点。这就是所谓的原型链。

另外在js中我们定义一个数组([])其实就是对构造函数Array进行实例化。我们定义一个对象({})，其实就是对构造函数Object进行实例化。
